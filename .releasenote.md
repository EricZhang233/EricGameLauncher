# Eric Game Launcher (EGL) **v1.0.9** — 一键入库，扫描进化

> [!IMPORTANT]
> **本次更新带来了全新的本地游戏自动扫描系统，支持一键批量导入，同时通过底层架构重构确保了 UI 与数据的实时强一致性。**
> **本次更新由于先前发现重大缺陷撤包，故存在紧急修复和功能重构，可能存在未被发现的bug，请根据本更新日志自由选择是否升级。**

>[!NOTE]
> **计划于近期发布第一个稳定版，稳定版将着重修复已发现的bug。**
> 稳定版将以优化和修bug为主，致力于构建一个尽量稳定的版本。
> 每个版本号结尾为[10]的倍数的版本(如x.x.10、x.x.20)将被视为稳定版，稳定版可能存在第四位版本号(如x.x.10.x)。
---

## 🔥 重磅新增：本地游戏自动扫描与一键导入

- **一键全量入库 (One-Click Import All)**：新增“导入全部新增项目”功能，扫描完成后一键即可将所有识别到的本地游戏入库，极大提升了库管理效率。
- **全域精准扫描 (Wide-Spectrum Scan)**：原生支持 Steam、Epic Games 及 Xbox (UWP) 三大平台已安装游戏的自动识别、路径提取与图标抓取。

---
## 🛠️ bug修复：多项bug和底层逻辑修复
- **主要修复：回退至本地解析Steam游戏图标**：由于Steam更新了本地库vdf文件导致无法解析，现在已回退至本地进行最大程度的基于库目录识别的图标提取。

---

## 🏗️ 架构革新：全状态标准化刷新系统

- **事件驱动 UI 同步**：引入 `ConfigService.DataChanged` 全局事件，将 UI 刷新逻辑从手动维护升级为架构级自动对齐。
- **全局刷新下沉**：核心逻辑（图标重建、平台校验）统一至 `Core.cs`。现在任何数据修改都会通过单一事件源同步 UI，消除了冗余代码。
- **性能优化与竞态消除**：优化了异步写入策略，在全局更新期间自动抑制冗余刷新，确保操作流畅无抖动。

---

## 🚀 细节优化与体验增强

- **受支持平台说明**：在扫描界面实时展示当前支持的平台信息，并关联了直观的操作引导图标。
- **存储模式切换加速**：重构了系统模式与便携模式切换后的刷新流程，加载提速明显。
- **全量国际化**：补全并优化了扫描、属性管理及排序对话框在 8 国语言下的翻译词条。
