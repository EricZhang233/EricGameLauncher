## はじめに

**Eric Game Launcher** は、ミニマリストのために作られた次世代のゲームランチャーです。従来のプラットフォームの肥大化を排除し、**WinUI 3** と **.NET 10** の強力なパフォーマンスを活用することで、ミリ秒単位の起動体験と Windows 11 ネイティブの視覚的な楽しみを提供します。

ゲームが Steam、Epic、または独立した exe ファイルであっても、ここがそれらの統一された拠点となります。

> [!IMPORTANT]
> **動作環境要求**: 本プログラムは **Windows 11 25H2** 以降のバージョンのみをサポートしています。

## 機能特性

### プレイヤーのために
*   **フルリンク実行サポート**:
    *   **多次元起動**: 「メインプログラム」、「マネージャー」、「代替起動」、「同時実行」、および「カスタム右クリックメニュー」をカバーする完全なライフサイクル管理。
    *   **全タイプ Run サポート**:
        *   **ウェブサポート**: ウェブリンクに直接対応し、ワンクリックで攻略サイトや公式サイトにアクセス可能。
        *   **ネイティブ EXE/LNK**: スペースを含む長いパスをサポートし、インテリジェントな引数分割エンジンを内蔵。
        *   **URL プロトコル**: `steam://`、`epic://`、`starward://` などのプロトコルに完全対応し、プラットフォーム級のインタラクションを実現。
        *   **Store アプリ**: `shell:AppsFolder\` を介したストアアプリの起動と、管理者権限での执行をサポート。
        *   **環境変数**: すべてのパスで `%AppData%`、`%LocalAppData%` などの標準環境変数の自動展開をサポート。
*   **究極の軽量化**：必要な時に起動し、使い終わったらすぐ終了。バックグラウンドサービスなし、広告なし、システムリソースの使用率はほぼゼロです。
*   **効率的な検索**：拼音フルスペル、拼音の頭文字、英語の頭文字、スペース区切りの頭文字、および大文字によるマッチ検索に対応。  
*   **プロフェッショナル級のプロパティ制御**：これは単なる簡素なショートカットではありません。
    *   **管理者として実行**：「ゲームメインプログラム」と「ゲームマネージャー」に対して個別に権限昇格を設定でき、権限不足で起動できない問題を解決します。
    *   **マネージャーの実行**：マネージャーのパスを設定することで、ゲームが起動するだけでなく、プラットフォームサービスも正しく呼び出されるようにします。
    *   **フルパラメータサポート**：設定されたすべての実行対象はエンドツーエンドの実行サポートを受けます。
    *   **メインプログラムの代替起動**：この項目をチェックすると、カスタムコマンド（例：`starward://`）を使用して、元の EXE の起動ロジックを完全に置き換えることができます。
    *   **起動時に同時実行**：ゲームをプレイする時に翻訳機、タイマー、またはパフォーマンスモニターを自動的に開きたいですか？これを設定すれば、ワンクリックで同時起動が可能です。
    *   **アイコンの変更**：カバーとして手動で画像をアップロードすることをサポート。
    *   **カスタム右クリックメニュー**：各プロジェクトに最大 10 個のカスタムメニュー項目を追加可能。
        *   タイトル、コマンドパス、引数を個別に設定可能。
        *   カスタム項目ごとに「管理者として実行」を設定可能。
        *   **スマート位移**：中間項目を削除すると、後続の項目が自動的に繰り上がり、リストをコンパクトに保ちます。
    *   **アイコンサイズ**：コンパクトなリストから没入感のある大きなタイルまで、自由自在に変更できます。
 *   **自動アップデート対応**：
    *   **内蔵アップデートチェック**: 起動時に GitHub Releases を自動確認し、常に最新バージョンをご利用いただけます。  
    *   **静かな通知**: アップデートが利用可能な場合、画面右上のバージョン番号が赤色になり、更新アイコンが表示されます — 煩わしいポップアップ通知はありません。  
    *   **ワンクリックダウンロード**: 新バージョンを検出した場合、ワンクリックでダウンロードとインストールが完了します。手動操作は不要です。  

### ギークのために
*   **完全ポータブルモード (Portable Mode)**：すべてのデータ（設定、キャッシュされたアイコン）をプログラムディレクトリ内に保存することをサポート。USB メモリに入れて、あなたのゲームライブラリを持ち運び、どの PC でも执行できます。
*   **シームレスなデータ移行**：システムインストールモードからポータブルモードへの切り替えも、ワンクリック移行ですべての設定とアイコンが自動的に移動し、再設定は不要です。
*   **スマートなアイコン抽出**：`.exe`、`.lnk`、さらには Steam/Epic の URL プロトコルを自動的に解析し、高解像度のアイコンを抽出してキャッシュします。

## 技術的なハイライト

開発者にとって、Eric Game Launcher は最小限のコードでいかに現代的な Windows デスクトップアプリを構築するかを示しています。

*   **最先端の技術スタック**：最新の **Windows App SDK (1.6+)** と **.NET 10** に基づいて構築されており、デスクトップアプリにおける WinUI 3 の大きな可能性を示しています。
*   **ネイティブパフォーマンス**：
    *   `P/Invoke` (User32.dll) を利用して、効率的なシステムレベルのアイコン抽出を実現。
    *   `System.Text.Json` を使用して、高性能でメモリ使用量の少ないデータシリアル化を実現。
*   **クリーンなアーキテクチャ**：
    *   **効率的なデータバインディング**：`ObservableCollection` と XAML バインディングにより、スムーズな UI インタラクションと明快で直感的なコードロジックを実現。
    *   **静的サービス設計**：`ConfigService` や `I18n` などのコアモジュールは静的クラスとして設計されており、オーバーヘッドゼロで呼び出せ、極めてシンプルかつ効率的です。
*   **モダンなビルド**：
    *   **Unpackaged リリース**：煩雑な MSIX パッケージングや証明書のインポートを廃止し、クリーンな `.exe` 実行ファイルを直接生成します。

## 既知の問題：
1. **一部の入力法との不互換性**: WinUI 3 フレームワークの低レベル入力チャネルの特性により、一部の入力法（WeChat 入力法など）で候補ウィンドウが正常に表示されない場合があります。**システム標準の Microsoft 入力法の使用を推奨します**、または他の主要な入力法。WinUI 3 は従来の小窓ハンドル (HWND) を廃止し、純粋な TSF (Text Services Framework) アーキテクチャを採用しました。WinUI 3 の**未パッケージ (Unpackaged) モード**では、WeChat 入力法などの一部の入力法が、基盤となる適応が不完全なためにカーソルの物理画面座標を正しく解析・取得できず、候補ウィンドウの描画をサイレントに断念します。この互换性の欠陥は、他の未パッケージの WinUI 3 アプリケーションにも存在します。これは入力法側の低レベルな適応不足によるものであり、Eric Game Launcher とは関係ありません。
